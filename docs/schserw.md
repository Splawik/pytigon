# schserw — Embedded Django Server Extensions

`schserw` is the Django application bundled inside the `pytigon` package.
It ships as the default `"schserw.schsys"` app in `INSTALLED_APPS` and
provides the system views, middleware, template tags, settings layering, and
the GraphQL schema that every pytigon project inherits.

This is not a user-facing app like those generated by `schdevtools` — it is
the runtime substrate that makes a pytigon project behave like a pytigon
project.

---

## Layout

```
schserw/
├── urls.py               # Root URL conf (auth, login, media, dialogs)
├── routing.py            # Django Channels (WebSocket) routing
├── settings/             # Layered settings
│   ├── base.py           # Base Django settings
│   ├── features.py       # Feature flags (auth, allauth, channels, prometheus)
│   ├── infra.py          # Infrastructure (DB, cache, hosts, allowed IPs)
│   └── __init__.py       # Composes base + features + infra via star-import
├── schsys/               # The system app
│   ├── app_manager.py    # AppManager — per-request app registry & menu
│   ├── context_processors.py  # sch_standard context (user, apps, client type)
│   ├── initdjango.py     # Runtime monkey-patches (BaseForm.as_p, widget_attrs)
│   ├── schema.py         # GraphQL Query/Mutation (graphene)
│   ├── views.py          # Auth, dialogs, plugin_template, static handlers
│   ├── cache_message_storage.py  # Cache-based Django messages backend
│   ├── remotebackend.py  # Remote backend hooks
│   └── templatetags/     # UI tag libraries
│       ├── exsyntax.py        # Main UI tags (field, Form, frame, icon, tree…)
│       ├── exsyntax_form.py   # Form-related tags
│       ├── exsyntax_include.py # Inclusion tags (frame, subform, jscript, css)
│       ├── exfiltry.py        # UI filters (only_items_containing, perms, …)
│       ├── htmlwidget.py      # {% html_widget %} block tag
│       ├── expr.py            # {% expr %} — inline Python expressions
│       ├── catch.py           # {% catch %} — capture rendered output
│       ├── collapse.py        # Whitespace collapsing
│       └── contenttypes.py    # ContentType helpers
├── schmiddleware/        # Custom middleware
│   ├── schjwt.py         # JWT-based user middleware (graphql-jwt)
│   ├── schpost.py        # ViewPost / view_post — POST logging
│   ├── csrf.py           # CSRF tweaks for embedded mode
│   ├── vary.py           # Vary header handling
│   └── whitenoise2.py    # WhiteNoise extensions
├── oauth2_ext/           # OAuth2 provider extensions
├── make-messages.py      # i18n: extract messages
├── compile-messages.py   # i18n: compile .po → .mo
```

## Settings Layering

Settings are composed by `schserw/settings/__init__.py`, which star-imports
`base`, `features`, and `infra`. The `finish(settings)` hook then applies
runtime adjustments (e.g. Prometheus middleware wiring).

| Layer | Responsibility |
|-------|----------------|
| `base.py` | Django defaults, `INSTALLED_APPS`, templates, static files |
| `features.py` | Optional features gated by env vars: `ALLAUTH`, `CHANNELS`, `PROMETHEUS_ENABLED`, `THREE_LEVEL_MENU`, `EMBEDDED` mode, etc. |
| `infra.py` | `DATABASES` (from `DATABASE_URL`), `CACHES`, `ALLOWED_HOSTS`, `SECRET_KEY`, static/media roots, `DEBUG` |

A project's `settings_app.py` typically does:

```python
from schserw.settings import *  # noqa
from schserw.settings import finish

# project-specific overrides…
finish(globals())
```

## AppManager

`schserw.schsys.app_manager.AppManager` is instantiated per request (via the
`sch_standard` context processor) and provides:

- `_get_apps(prj)` — cached list of installed pytigon apps with their titles,
  permissions, URLs, and module metadata. Cached at module level on
  `(prj, host)` so it does not leak request/user references.
- `get_app_items(prj)` — flattened URL/menu items per app.
- `get_apps_width_perm(prj)` / `get_app_items_width_perm(prj)` —
  permission-filtered variants for the current user.
- `appname()` / `appid()` — resolve the current app from the request path.
- `login_providers()` — list of configured allauth providers (if `ALLAUTH`).

## Context Processors

`scherw.schsys.context_processors.sch_standard` is the main context
processor. It injects:

- `client_type` / `browser_type` — device detection (desktop / tablet /
  smartfon / hybrid), based on `HTTP_USER_AGENT` and `HTTP_ACCEPT`.
- `apps` — `AppManager(request)` instance (lazy).
- `is_embedded` / `is_hybrid` — runtime mode flags.
- `LANG` / `PRJ_NAME` / `MAIN_TITLE` — project metadata.

Device detection uses a compiled regex over `MOBILE_USER_AGENTS`, so it is
O(1) per request rather than a linear substring scan.

## Template Tags

The `exsyntax` library is the single largest UI surface in pytigon. It
provides tags that render forms, tables, frames, icons, trees, paginators,
and dialog triggers. Highlights:

| Tag | Purpose |
|-----|---------|
| `{% field form.field %}` | Render a form field with Pytigon layout |
| `{% Form form %}` / `{% vert_form %}` / `{% col2_form %}` | Form layouts |
| `{% frame href height %}` | Embed another URL in an iframe-like panel |
| `{% subform name %}` | Embed a sub-form within the current form |
| `{% icon class_str width height %}` | Render an SVG/Font Awesome icon |
| `{% editable name title url %}` | Inline-editable field |
| `{% paginator %}` / `{% sorted_column %}` | Table pagination / sortable headers |
| `{% comboselect %}` | Combo-box select widget |
| `{% do_tree %}` | Render a tree widget from a model hierarchy |
| `{% subtemplate template_string %}` | Render a Django template from a string |
| `{% do_html_widget %}` | Render a registered HTML widget |
| `{% include_wiki wiki_str %}` | Embed wiki content |
| `{% markdown2html markdown_str %}` | Render Markdown to HTML |

Filters in `exfiltry` include `only_items_containing`, `user_can_change_password`,
`prefetch_related`, and many permission/object helpers.

## Middleware

| Middleware | Purpose |
|------------|---------|
| `schjwt.JWTUserMiddleware` | Resolves the user from a `graphql-jwt` token; caches the user on the request. |
| `schpost.ViewPost` / `view_post` | Logs POST bodies at DEBUG level (dev aid). |
| `csrf` | Relaxes CSRF for the embedded `127.0.0.2` client when `EMBEDDED` mode is on. |
| `vary.VaryMiddleware` | Adds `Vary` headers for client-type-aware caching. |
| `whitenoise2` | WhiteNoise integration with pytigon's static path resolution. |

## GraphQL Schema

`scherw.schsys.schema` exposes a graphene schema with:

- `PublicQuery` / `PublicMutation` — unauthenticated surface
  (`resolve_users` returns only public fields).
- `Query` / `Mutation` — authenticated surface (full user info, app data).

The schema is wired into `urls.py` via `graphql_jwt` when `GRAPHQL` is
enabled in `features.py`.

---

## URL Surface

`scherw.urls` provides:

- `/sch_login/`, `/sch_logout/` — auth endpoints (CSRF-exempt for embedded)
- `/site_media/<path>` — protected media (per-user folder isolation)
- `/dstatic/<path>` — dynamic static (compiled from ihtml on demand)
- `/datedialog/`, `/listdialog/`, `/treedialog/`, `/tabdialog/` — dialog
  views invoked from the GUI
- `/plugin_template/<template_name>` — render a plugin template server-side
- `/api/schema/` — GraphQL endpoint (when enabled)

## i18n

Three scripts drive translations:

- `make-messages.py` — runs `xgettext` / `makemessages` to extract strings
- `translate.py` — orchestrates per-locale processing
- `compile-messages.py` — compiles `.po` to `.mo`

The bundled `ext_lib/pygettext.py` is a vendored copy of CPython's gettext
extraction tool, used by `make_i18n.py` at the repo root.

## Runtime Monkey-Patches

`scherw.schsys.initdjango` applies targeted patches to Django internals
at import time (guarded by a `RuntimeWarning` if the Django version differs
from the targeted release). Current patches:

- `BaseForm._html_output` — left-aligned headers
- `BaseForm.as_p` — render via `django_bootstrap5`
- `CharField.widget_attrs` — inject `max_length` / `size` attributes
- `models.TreeForeignKey` / `models.GTreeForeignKey` — aliases to
  `ForeignKey` for compatibility with mptt/django-tree-libs
- `db.models.fields.prep_for_like_query` — backslash escaping fix
